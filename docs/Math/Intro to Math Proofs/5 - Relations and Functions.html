Relations and functions

Relations

  • R⊆A×BR \subseteq A \times BR⊆A×B
    • R contains relationships between A and B
  • (a,b)∈R(a,b) \in R(a,b)∈R, which can be denoted as aRbaRbaRb
    • ex: in a relationship defined by x>yx>yx>y, 5>35>35>3 is true, so 5R35R35R3, given 5∈A∧3∈B5 \in A \land 3 \in B5∈A∧3∈B
  • Types of relationships in A×AA \times AA×A
    • reflexive: ∀x∈A,xRx\forall x \in A, xRx∀x∈A,xRx
      • every element in A relates to itself
      • rules such as =,∣,⊆,≤,≥,≡=, |, \subseteq, \leq, \geq, \equiv=,∣,⊆,≤,≥,≡
    • symmetric: ∀x,y∈A,xRy  ⟹  yRx\forall x, y \in A, xRy \implies yRx∀x,y∈A,xRy⟹yRx
      • if one element relates to another element, that other element must also relate back to the first element
      • rules such as =,≠,≢,≡,=, \neq, \not\equiv, \equiv,=,=,≡,≡,
    • transitive: ∀x,y,z∈A,xRy∧yRz  ⟹  xRz\forall x, y, z \in A, xRy \land yRz \implies xRz∀x,y,z∈A,xRy∧yRz⟹xRz
      • if x relates to y, and y relates to z, then x must also relate to z
      • rules such as <,>,≤,≥,∣,∤<, >, \leq, \geq, |, \nmid<,>,≤,≥,∣,∤
  • Equivalence relation: a relationship that's reflexive, symmetrical, and transitive
  • Given an equivalent relation R, {x∈A:xRa}\{x \in A : xRa\}{x∈A:xRa} is a set of all elements that relate specifically to aaa
    • given a rule x≡a(mod3)x \equiv a (mod 3)x≡a(mod3), when aaa is 2, a set can be made for all x'es that when divided by 3, share the same remainder as 2 when divided by 3
    • That set can be denoted [a][a][a]
    • mod3mod 3mod3 can result in 3 possible remainders: 0, 1, 2, so its partitions are [0], [1], [2].
      • [3] is the same thing as [0], because 3 divided by 3 has the same remainder as 0 divided by 3, so multiple representatives are possible

Functions

  • f:A→Bf : A \rightarrow Bf:A→B is a relationship f⊆A×Bf \subseteq A \times Bf⊆A×B
  • ∀a∈A,(a,b)∈f\forall a \in A, (a,b) \in f∀a∈A,(a,b)∈f
    • each input has an output
  • (a,b)∈f∧(a,c)∈f  ⟹  b=c(a,b) \in f \land (a,c) \in f \implies b = c(a,b)∈f∧(a,c)∈f⟹b=c
    • each input has only one corresponding output
  • AAA is the domain, or the set of inputs
  • BBB is the codomain, or the set of possible outputs
  • The range is {b=f(a):a∈A}\{b = f(a) : a \in A\}{b=f(a):a∈A}, or the actual outputs values that are used
  • Types of functions
    • injective (1 to 1): ∀x,y∈A,f(x)=f(y)  ⟹  x=y\forall x, y \in A, f(x)=f(y) \implies x=y∀x,y∈A,f(x)=f(y)⟹x=y
      • each output has only one corresponding input
    • surjective (onto): ∀b∈B,∃a∈A,f(a)=b\forall b \in B, \exists a \in A, f(a) = b∀b∈B,∃a∈A,f(a)=b
      • every element of the codomain is in the range
    • bijective: both injective and surjective
    • inverse: a function takes an input and gives a output. its inverse takes that output, and turns it back into the input. requires the function to be bijective

Pigeonhole principle

  • given f:A→Bf : A \rightarrow Bf:A→B
  • if ∣A∣>∣B∣|A| > |B|∣A∣>∣B∣, f is not injective
  • if ∣A∣<∣B∣|A| < |B|∣A∣<∣B∣, f is not surjective
